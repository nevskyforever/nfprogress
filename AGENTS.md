## О проекте

Это настольное приложение на Python с графическим интерфейсом PySide6 для писателей, чтобы отслеживать прогресс в написании текстов, с игровыми механиками (опционально).

Приложение собирается через Nuitka для:

* Windows;
* macOS Apple Silicon;
* macOS Intel.

## Основные правила работы

* Выполняй только задачу, указанную пользователем.
* Не изменяй несвязанные части проекта.
* Не проводи полный рефакторинг без прямого запроса.
* Перед изменением кода найди связанные файлы и места использования.
* Для поиска по проекту используй `rg`.
* Не изучай весь проект, если задача относится к отдельному модулю.
* Сохраняй существующую архитектуру и стиль кода.
* Не добавляй новые зависимости без необходимости.
* Не меняй публичное поведение приложения без прямого запроса.

## Работа с файлами

* Сначала определи минимальный набор файлов, необходимых для задачи.
* Не открывай большие файлы целиком, если достаточно найти нужную функцию или класс.
* Не изменяй файлы сборки, зависимости и конфигурацию без необходимости.
* Не изменяй автоматически сгенерированные файлы.
* Не удаляй код, пока не проверено, что он нигде не используется.

## Python

* Используй возможности текущей версии Python проекта.
* Добавляй аннотации типов там, где они улучшают понимание кода.

* Сохраняй понятные имена функций, классов и переменных.

## PySide6

* Не выполняй долгие операции в главном потоке интерфейса.
* Не блокируй интерфейс сетевыми запросами, ожиданием или тяжёлыми вычислениями.
* Соблюдай существующую систему сигналов и слотов.
* Не меняй структуру интерфейса без прямого запроса.
* Учитывай работу приложения на Windows и macOS.

## Сборка

* Не запускай полную сборку Nuitka без необходимости.
* Не запускай одновременно сборки для всех платформ, если задача этого не требует.
* При изменении сборочных скриптов проверяй различия между Windows, macOS ARM и macOS Intel.
* Не изменяй пути публикации, SSH-настройки, ключи и данные сервера без прямого запроса.

После изменений:

1. Проверь синтаксис затронутых Python-файлов.
2. Запусти только относящиеся к задаче тесты.
3. Проверь импорты изменённых модулей.
4. При возможности проверь изменённую функцию отдельно.
5. Не запускай весь набор тестов, если достаточно точечной проверки.

Если тестов для изменённого кода нет, сообщи об этом.
